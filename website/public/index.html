Client-side key derivation

PBKDF2 with 100,000 iterations derives your AES key locally. Firebase receives only ciphertext.

🛡️

Memory-safe sessions

The encryption key lives only in sessionStorage, auto-wiped on lock or 15-minute timeout.

🌐

Firestore security rules

Every user can only access their own vault document. Not even admins can read your data.

🎲

True randomness

All generation uses crypto.getRandomValues() — never Math.random().

// KeyNova — zero-knowledge flow
const pw = getUserInput()
// never stored or sent to server
const salt = crypto.getRandomValues(16)
const key = await PBKDF2(pw, salt, 100000)
// 100,000 iterations, SHA-256
const blob = await AES_GCM.encrypt(key, vault)
await Firestore.set("vaults/uid", blob)
// Firebase only ever sees ↑ blob
